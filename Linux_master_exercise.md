### 리눅스 시스템 이해

* 리눅스에서 인식되는 장치 파일명의 종류가 다른것?

  * S-ATA, USB 메모리, SCSI 는 sdx 로 인식
  * IDE디스크, 그냥 ATA 는 hdx 로 인식

* RAID 

  * 대표 기술 : 스트라이핑, 미러링
  * 사용 중 디스크 추가 용량 확장 불가
    * 하지만 전원이 켜져있는 상태에서 하드드라이브 교체 할 수 있음. Hot Swap Bay 기술
  * 오류 검사를 위해 패리티나 ECC 사용
  * 소프트웨어 방식과 하드웨어 방식이 존재
  * RAID -5 
    * 최소 디스크 3개 이상 구성
    * 디스크 오류 2개 이상 결함 생길 경우 손실 발생가능
    * 1GB 하드디스크 4개로 구성하면 사용 가능한 용량은 3GB, 1GB는 패리티 (1/N 이 패리티로 쓰임.)
    * N 값이 높아질수록, 더 많은 디스크를 쓸 수록 효율이 높아짐.
  * RAID-6
    * 2GB 6개로 구성했다면? 가용 공간은
    * 2차 패리티를 구성하기 때문에 6개중 2개가 패리티 로 사용되고, 4개의 디스크가 가용가능하다. 2*4 = 8GB 사용가능

* LVM : 여러 개의 디스크를 하나로 묶어서 사용하는 기술, 파티션의 크기를 사용중에 늘리거나 줄일 수 있다.

  * 다른 크기의 하드디스크로도 구성 가능
  * 파티션 확장 시 데이터 이전 작업 필요 없음
  * PE ? Physical Extension :  일종의 블록같은 영역, 보통 1단위당 4MB

* root 의 패스워드를 잊어버려 GRUB 을 통해 싱글모드로 접근하려함.

  GRUB 의 가장 쉬운 모드? [a]

* GRUB 설정 timeout=10 에 대한 설명?

  * 10초 동안 대기한 후에 기본 설정된 운영체제로 부팅한다.

### 네트워크의 이해

* LAN 구성방식

  * STAR 형 = 중앙집중형
    * 중앙의 제어기를 중심으로 모든 기기를 Point-To-Point 방식으로 연결
    * 단말기 추가 및 오류 진단이 용이
    * 중앙 컴퓨터 고장 시 전체 네트워크에 영향
    * 설치비용이 많이 들고, 회선 수가 증가하면 제어 복잡
  * BUS 형 : 하나의 통신회선에 여러 컴퓨터를 연결, 모든 장치들을 동등한 조건으로.
    * 컴퓨터의 수에 따라 성능 좌우, 종단기 존재
  * MAC 주소를 기반으로 상호간의 데이터를 주고 받으며 BNC, UTP 케이블 사용 = Ethernet

* MAN 을 위한 국제 표준안 IEEE802.6 에 정의되어 있는 것?

  * DQDB

* 회선 교환 방식?

  * 송수신 호스트 간에 데이터 전송을 위한 연결 경로를 설정한 후 전송
  * 고정된 대역폭을 할당받아 전송
  * 모든 데이터가 같은 경로로 전송된다
  * 안정적인 데이터 전송률 지원

* 패킷 교환 방식

  * 전송 대역폭이 동적
  * 이론상 호스트의 무제한 수용이 가능
  * 모든 데이터가 같은 경로로 전송되지 않을 수도 있다.
  * 패킷마다 오버헤드 비트가 존재
    * 오버헤드? : 원래 처리하려는 시간에 비해 더 걸린 시간.

* 이더넷 매체 표기법

  10 = 10Mbps, 100 = 100Mbps, 1000 = 1Gbps, 

  2,5  = 동축, Coaxial Cable,  2= 200m 5 = 500m

  T = UTP Unshielded Twist Pair 

  F = Optical Fiber Cable 광케이블

  * 10BASE-5 : 10Mbps 전송 속도 , -5 = 500M 동축케이블
  * 100BASE-TX : 100Mbps 전송 속도, UTP-5 zpdlqmf
  * 1000BASE-LX : 장파장 광섬유 1000Mbps
  * 100BASE-FX: 100Mbps, 광케이블

* ATM : Cell Relay 

  * 53Byte의 고정 길이 패킷을 이용하여 순서대로 자료를 전송하는 방식

  * 흰주, 주, 흰녹, 파, 흰파, 녹 , 흰갈, 갈
